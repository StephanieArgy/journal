Also, there was this swell little cheatsheet, which if it were on paper would now be all tattered and dog-eared from being used so much:

Workflow

Do your work on your branch. When you are ready to share your work with the team, do the following steps:

  • switch to the master branch ("git checkout master")
  • pull from github to bring in your team's work ("git pull")
  • switch to your branch ("git checkout mybranch")
  • merge from master to your branch ("git merge master")
  • resolve any merge conflicts that may happen using your editor
  • switch to the master branch ("git checkout master")
  • merge from your branch to master ("git merge mybranch")
  • push your changes to github ("git push")

Best. Thing. Ever.

Primer Pro: Fifth Week

Saturday, March 7, 2015: Start of Primer Pro

Our first JavaScript/jQuery assignment starts with our having to make a repository and establish a remote on github.com. "This should be review," say the instructions.

So here's my review -- a knucklehead cheatsheet:

To Create a New Project:

  1. On GitHub, make a new repository.
  2. In Terminal, navigate to the directory that will contain the local directory for that respository.
  3. In that directory: git clone https://github.com/MyGitHubName/repositoryname.git.
  4. In Terminal, go into newly created local repo clone using cd.
  5. Back on GitHub, create a new branch called gh-pages.
  6. In Terminal (and in the new directory): git fetch origin gh-pages. (At this point, only master branch will be visible with git branch.)
  7. In terminal: git checkout gh-pages. (Now git branch will show both branches.)
  8. To start working -- in Terminal: git checkout master.

After working, to add changes and update gh-pages:

  1. In terminal, check status, see what's changed: git status.
  2. Add changes: git add (followed by files that have changed or added, one git add command for each)
  3. Commit changes: git commit -m"Message describing this set of changes"
  4. Push changes to master branch online: git push.
  5. Change to gh-pages branch: git checkout gh-pages.
  6. Merge content from master branch: git merge master.
  7. Update online version of gh-pages: git push.

And as long as I'm scribbling down helpful tidbits, here are some resources for JavaScript:

  • jQuery.com
  • JavaScript documentation: http://api.jquery.com/
  • http://blog.teamtreehouse.com/beginners-guide-to-jquery

Primer Pro: Seventh Week

Bootstrap

I may have to rethink my system of naming articles by week and day, because clearly there's been a breakdown...

And, speaking of breakdowns -- JavaScript, jQuery and I had some big troubles this last week. Part of it was bad timing: a jam-packed, non-stop weekend coinciding with the most complex subject matter so far. I needed more time to figure out some of the basic concepts of JavaScript and jQuery, and that didn't happen. As a result, I felt stranded in the land of Cargo Cult, flapping my arms in the direction of the code, with little understanding of what anything meant. When my partner Sharon and I got together, I was frustrated by how little of substance I could contribute to our assignment.

I know that part of the challenge of programming is figuring out how to do things that we don't yet know how to do, but I didn't feel as though I had the foundational knowledge to truly figure things out yet. It may be the nature of the course, because it has to move so quickly to fit in all the subject matter in such a short period of time -- or maybe it was just me.

We were also hamstrung by weird, glitchy things that kept happening, like not knowing that we couldn't start directories with an underscore in Github. And, as so often can be the case, those glitches were contagious, so easy things that should have taken five minutes (like making a two-column layout for larger screens) just didn't work. Why? I still don't know.

But...I'm not giving up hope! I did set aside JavaScript and jQuery since Monday's class, so that I could focus on understanding the basics of Bootstrap before our next assignment. This weekend, though, is as free as last one's was frenetic, so JavaScript and jQuery and I will be spending some substantial time together and trying to reach an understanding.

Stephanie Argy, April 15, 2015.
